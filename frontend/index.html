The frontend uses a NodePort service so you can access it from outside the cluster. The backend uses a ClusterIP service, reachable only within the cluster. The frontend's nginx reverse-proxies /api/ requests to the backend service. Try scaling the backend deployment and refreshing to see different pod hostnames!
